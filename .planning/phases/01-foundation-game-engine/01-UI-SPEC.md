# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation & Game Engine phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Scope: project scaffold + fully playable local hot-seat game. No multiplayer UI, no animations, no end-game modal — those are Phase 3. This contract defines the game board, player areas, score display, and page layout needed for Phase 1 to be visually coherent and playable.

---

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between dot labels, inline icon padding |
| sm | 8px | Padding inside score chips, gap between score number and label |
| button | 16px | Vertical padding on buttons (`py-4`) |
| md | 16px | Default element padding (player cards, page content padding) |
| lg | 24px | Section padding inside side panels |
| xl | 32px | Gap between side panels and center board column |
| 2xl | 48px | Vertical padding above/below the 3-column layout |
| 3xl | 64px | Not used in Phase 1 |

Exceptions: SVG board uses its own internal coordinate space (DOT_SPACING = 80px, PADDING = 40px within a 400×400 viewBox). These are SVG user-unit values, not CSS layout values, and do not need to conform to the 8-point scale.

Touch targets: not applicable (desktop-first, no mobile requirement in Phase 1).

## Color

| Role | Hex | Tailwind Class | Usage |
|------|-----|----------------|-------|
| Dominant (60%) | #F8FAFC | `bg-slate-50` | Page background, board background surface |
| Secondary (30%) | #FFFFFF | `bg-white` | Player cards, panel backgrounds |
| Accent P1 (blue) | #3B82F6 | `bg-blue-500` / `text-blue-500` | Player 1 box fills (opacity-40), Player 1 score card active ring, Player 1 name text when active |
| Accent P2 (red) | #EF4444 | `bg-red-500` / `text-red-500` | Player 2 box fills (opacity-40), Player 2 score card active ring, Player 2 name text when active |
| Dot & drawn line | #1E293B | `fill-slate-800` / `stroke-slate-800` | SVG dots and drawn line strokes |
| Undrawn line | #E2E8F0 | `stroke-slate-200` | Idle undrawn line segments (default invisible state) |
| Hover line | #94A3B8 | `stroke-slate-400` | Hovered undrawn line (pre-click highlight) |
| Destructive | #EF4444 | `text-red-500` | Player 2 color is reused; no separate destructive action exists in Phase 1 |

### 60/30/10 split

- 60% dominant: `slate-50` (#F8FAFC) — fills the page background and board area
- 30% secondary: `white` (#FFFFFF) — fills player cards and panel containers
- 10% accent: blue (#3B82F6) and red (#EF4444) — reserved for: Player 1 box fills, Player 2 box fills, active player card ring, active player name color, and the turn status label color

Accent reserved for (explicit list):
1. SVG box fill — semi-transparent player color at `opacity-40` on claimed boxes
2. Active player card — `ring-2 ring-blue-400` (P1) or `ring-2 ring-red-400` (P2) on the active player's score card
3. Turn label text — colored with the active player's accent color
4. Nothing else — drawn lines use `slate-800`, hover lines use `slate-400`, dots use `slate-800`

> Source: design_direction — "Player 1 = blue (#3B82F6), Player 2 = red (#EF4444). Board background: white or very light gray."

## Component Inventory

### 1. Page Layout (`app/page.tsx` + `app/game/page.tsx`)

**Home screen** (`/`):
- Centered single-column layout, `max-w-sm`, vertically centered in viewport
- "Dots & Boxes" display heading (28px, weight 600)
- Username input: `border border-slate-200 rounded-lg px-4 py-2 text-base w-full`
- "Start Game" button (see CTA spec below)

**Game screen** (`/game`):
- Full-viewport height: `min-h-screen bg-slate-50`
- 3-column grid: `grid grid-cols-[240px_1fr_240px] gap-8 items-start px-8 py-12 max-w-[960px] mx-auto`
- Left column: Player 1 card (self)
- Center column: `flex flex-col items-center gap-4` — turn label + SVG board
- Right column: Player 2 card (opponent)

### 2. Player Card (`components/game/PlayerCard.tsx`)

States: active (current turn) | inactive (waiting)

**Active state:**
```
bg-white rounded-2xl p-6 ring-2 ring-blue-400   ← P1 or ring-red-400 for P2
```

**Inactive state:**
```
bg-white rounded-2xl p-6 opacity-60
```

Contents:
- Player name: `text-base font-normal text-slate-700`
- Score number: `text-xl font-semibold` — colored with player accent (`text-blue-500` or `text-red-500`)
- Score sub-label: `text-sm text-slate-400` — copy: "boxes"

### 3. Turn Label (`components/game/TurnLabel.tsx`)

Displayed above the SVG board, centered.

- Font: `text-base font-normal`
- Active player text color: `text-blue-500` (P1) or `text-red-500` (P2)
- Copy: "Your turn" when it is the local player's turn; "Player 1's turn" / "Player 2's turn" for hot-seat

### 4. SVG Game Board (`components/game/GameBoard.tsx`)

Container: `<svg viewBox="0 0 400 400" className="w-full max-w-[480px]">`

The SVG renders at a maximum CSS width of 480px (the viewBox is 400×400 user units; CSS scaling handles display size). Board background is transparent — page `bg-slate-50` shows through.

**Dots:**
- `<circle r={6} fill="#1e293b" />` — `slate-800`
- 5×5 grid at positions: cx = 40 + col×80, cy = 40 + row×80

**Undrawn lines (idle):**
- `stroke="#e2e8f0"` (`slate-200`), `strokeWidth={6}`, `strokeLinecap="round"`
- Invisible hit area: `stroke="transparent" strokeWidth={20} className="cursor-pointer"`

**Undrawn lines (hover):**
- `stroke="#94a3b8"` (`slate-400`) via Tailwind `group-hover` or `onMouseEnter`/`onMouseLeave` — the visible line switches to hover stroke on pointer enter
- Implementation: use `useState` local to each line group or a CSS `[&:hover>line]:stroke-slate-400` selector on the `<g>` wrapper

**Drawn lines:**
- `stroke="#1e293b"` (`slate-800`), `strokeWidth={6}`, `strokeLinecap="round"`
- No animation in Phase 1 — stroke-dashoffset animation is Phase 3

**Claimed boxes:**
- `<rect fill="#3b82f6" opacity={0.4} />` for P1, `fill="#ef4444"` for P2
- Geometry: x = 40 + col×80 + 6, y = 40 + row×80 + 6, width = 68, height = 68 (DOT_SPACING - DOT_RADIUS×2 = 80 - 12 = 68)
- No animation in Phase 1 — scale pulse is Phase 3

**Disabled state (GAME-06):**
- `pointer-events-none` class applied to the SVG wrapper when it is not the local player's turn
- In hot-seat Phase 1, this means the board is always interactive (both players use same browser) — the `disabled` prop is wired but its value is always `false` in Phase 1

**Render order (back to front):**
1. Box fill `<rect>` elements (behind everything)
2. Horizontal line `<g>` elements
3. Vertical line `<g>` elements
4. Dot `<circle>` elements (rendered last, always on top)

### 5. Start Button / CTA

- `bg-slate-800 text-white text-base font-semibold rounded-lg px-6 py-4 w-full hover:bg-slate-700 transition-colors`
- No loading state in Phase 1 (local game, instant start)

### 6. Score Bar (optional Phase 1 shorthand)

If a dedicated `ScoreBar.tsx` header is added above the board: `flex items-center justify-between w-full max-w-[480px] mb-2`. Shows "P1: {score}" and "P2: {score}" in their respective accent colors at `text-base font-semibold`. This is a stretch goal for Phase 1 — the player cards in the side columns are the primary score display.

## States and Interactions

### Line States

| State | Stroke | Width | Cursor | Hit area |
|-------|--------|-------|--------|----------|
| Undrawn idle | `#e2e8f0` (slate-200) | 6 | `default` | 20px transparent overlay |
| Undrawn hover | `#94a3b8` (slate-400) | 6 | `pointer` | 20px transparent overlay |
| Drawn | `#1e293b` (slate-800) | 6 | `default` | none (no click handler) |

### Board States

| State | Class on SVG wrapper | Trigger |
|-------|----------------------|---------|
| Interactive | (no class) | `disabled=false` |
| Locked | `pointer-events-none` | `disabled=true` |

### Player Card States

| State | Classes | Trigger |
|-------|---------|---------|
| Active (P1) | `bg-white rounded-2xl p-6 ring-2 ring-blue-400` | `currentTurn === 'p1'` |
| Active (P2) | `bg-white rounded-2xl p-6 ring-2 ring-red-400` | `currentTurn === 'p2'` |
| Inactive | `bg-white rounded-2xl p-6 opacity-60` | Not current turn |

### Game Over State (Phase 1 minimal)

Phase 3 delivers the polished end-game modal. Phase 1 minimum: when `gameState.status === 'finished'`, display a plain text banner above the board: `text-base font-semibold text-slate-800` reading "Player 1 wins!" / "Player 2 wins!" / "It's a draw!". Board becomes non-interactive (both pointer-events-none). A plain "Play Again" button resets state.

## Phase Scope Boundary

The following UI elements are explicitly OUT OF SCOPE for Phase 1. Do not implement them.

| Element | Phase |
|---------|-------|
| Line draw stroke-dashoffset animation (~200ms) | Phase 3 |
| Box fill scale-up pulse animation | Phase 3 |
| Game end modal with backdrop blur | Phase 3 |
| Emoji reaction panel and float animation | Phase 4 |
| Predefined chat buttons and message log | Phase 4 |
| Player presence indicator (green/yellow/red dot) | Phase 2 |
| Live score bar at top (as header bar) | Phase 3 |

Phase 1 game-over UX is intentionally minimal — a text banner and a reset button. No modal, no animation.
