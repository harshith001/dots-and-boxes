SVG structure (render order back-to-front per UI-SPEC):

1. **Box fills** -- `gameState.boxes.map()` rendering `<rect>` for each non-null box. P1 fill: `#3b82f6` opacity 0.4. P2 fill: `#ef4444` opacity 0.4. Use `boxRect(r, c)` for coordinates.

2. **Horizontal lines** -- `gameState.hLines.map()` with 5 rows x 4 cols. Each line is a `<g>` group:
   - Visible line: `stroke={drawn ? '#1e293b' : '#e2e8f0'}`, `strokeWidth={6}`, `strokeLinecap="round"`. Use `hLineCoords(r, c)`.
   - Hover state for undrawn lines: Use `useState` for `hoveredLine` (string key like `h-0-0`) or use CSS approach with `group` class on `<g>` and `[&:hover>line:first-child]` Tailwind selector. On hover, change stroke to `#94a3b8` (slate-400).
   - Hit area for undrawn lines: Invisible `<line>` with `stroke="transparent"` `strokeWidth={20}` and `className="cursor-pointer"`.
   - Click handler: `onClick={() => !drawn && onMove({ type: 'h', row: r, col: c })}` on the `<g>`.

3. **Vertical lines** -- `gameState.vLines.map()` with 4 rows x 5 cols. Same pattern as horizontal lines but using `vLineCoords(r, c)` and `type: 'v'`.

4. **Dots** -- 5x5 grid of `<circle>` elements. `r={6}`, `fill="#1e293b"`, coordinates from `dotPosition(r, c)`.

SVG container: `<svg viewBox="0 0 400 400" className={...}>` where className includes `w-full max-w-[480px]` and conditionally `pointer-events-none` when `disabled` is true. Use `gridToViewBox(5)` for the 400x400 values.

Hover implementation approach: Use a `hoveredLine` state (`string | null`) tracking the line key. On `onMouseEnter` of the `<g>`, set it. On `onMouseLeave`, clear it. Use the hovered state to set stroke to `#94a3b8` instead of `#e2e8f0` for the hovered undrawn line.

**Create `frontend/app/game/page.tsx` (Game screen):**

This MUST be a client component (`'use client'`). It wires the Zustand store to the UI components.

```tsx
'use client';
import { useEffect } from 'react';
import { useGameStore } from '@/store/gameStore';
import { GameBoard } from '@/components/game/GameBoard';
import { PlayerCard } from '@/components/game/PlayerCard';
import { TurnLabel } from '@/components/game/TurnLabel';
import Link from 'next/link';
```

Implementation:
1. Extract state from store: `const { gameState, startGame, makeMove, resetGame } = useGameStore()`
2. Auto-start game on mount: `useEffect(() => { if (!gameState) startGame(); }, [])` -- only call startGame if gameState is null to avoid double-invoke in StrictMode resetting the game.
3. If `gameState` is null, show loading or empty state.
4. 3-column layout per UI-SPEC:
   - Outer: `<div className="min-h-screen bg-slate-50">`
   - Grid: `<div className="grid grid-cols-[240px_1fr_240px] gap-8 items-start px-8 py-12 max-w-[960px] mx-auto">`
   - Left column: `<PlayerCard player="p1" name="Player 1" score={gameState.scores.p1} isActive={gameState.currentTurn === 'p1'} />`
   - Center column: `<div className="flex flex-col items-center gap-4">`
     - `<TurnLabel currentTurn={gameState.currentTurn} />` (only when game is active)
     - Game over banner (when `gameState.status === 'finished'`): `<p className="text-base font-semibold text-slate-800">` with text: "Player 1 wins!" / "Player 2 wins!" / "It's a draw!" based on `gameState.winner`
     - `<GameBoard gameState={gameState} onMove={makeMove} disabled={gameState.status !== 'active'} />`
     - Play Again button (when finished): `<button onClick={resetGame} className="bg-slate-800 text-white text-base font-semibold rounded-lg px-6 py-4 hover:bg-slate-700 transition-colors">Play Again</button>`
     - Back to home link: `<Link href="/" className="text-sm text-slate-400 hover:text-slate-600 transition-colors">Back to Home</Link>`
   - Right column: `<PlayerCard player="p2" name="Player 2" score={gameState.scores.p2} isActive={gameState.currentTurn === 'p2'} />`

5. The `disabled` prop on GameBoard: In Phase 1 hot-seat, the board is always interactive during active games (both players share same browser). Set `disabled={gameState.status !== 'active'}` so the board locks ONLY when the game is finished. The GAME-06 `pointer-events-none during opponent's turn` is a Phase 2 concern -- the prop is wired and functional but not triggered in hot-seat.
