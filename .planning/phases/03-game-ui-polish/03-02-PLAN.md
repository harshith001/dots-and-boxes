**Line draw animation** — when a line transitions from `drawn=false` to `drawn=true`, animate it.

Track which lines were drawn in the previous render using a `useRef<Set<string>>`. When a line is newly drawn (in current render but not in ref), apply the animation class.

```typescript
const drawnLinesRef = useRef<Set<string>>(new Set());

// In renderLine, detect newly drawn:
const isNewlyDrawn = drawn && !drawnLinesRef.current.has(key);
if (drawn) drawnLinesRef.current.add(key);
```

For the animated drawn line, use `strokeDasharray` + `strokeDashoffset`:

The line length can be computed from coords: `Math.hypot(x2-x1, y2-y1)`.

When `isNewlyDrawn`:
- Add `strokeDasharray={lineLength}` and `strokeDashoffset` to 0 via CSS animation
- Apply `className="animate-line-draw"` with `style={{ '--line-length': lineLength } as React.CSSProperties}`

```tsx
const lineLength = Math.hypot(coords.x2 - coords.x1, coords.y2 - coords.y1);

// In the drawn line's <line> element:
{drawn && (
  <line
    x1={coords.x1} y1={coords.y1}
    x2={coords.x2} y2={coords.y2}
    stroke={P2_COLOR} // We don't track who drew it — use a neutral drawn color
    strokeWidth={5}
    strokeLinecap="round"
    strokeDasharray={lineLength}
    strokeDashoffset={0}
    className={isNewlyDrawn ? 'animate-line-draw' : ''}
    style={isNewlyDrawn ? { '--line-length': lineLength } as React.CSSProperties : {}}
  />
)}
```

**Important**: The `useRef` approach will correctly track drawn lines across renders. Initialize the ref from the initial gameState:
```typescript
// Initialize ref from current drawn state on mount
useEffect(() => {
  gameState.hLines.forEach((row, r) => row.forEach((drawn, c) => {
    if (drawn) drawnLinesRef.current.add(`h-${r}-${c}`);
  }));
  gameState.vLines.forEach((row, r) => row.forEach((drawn, c) => {
    if (drawn) drawnLinesRef.current.add(`v-${r}-${c}`);
  }));
}, []); // eslint-disable-line react-hooks/exhaustive-deps
```

**Box claim animation** — the existing `className="animate-box-claim"` on box `<g>` elements is correct from Phase 2. Since SVG `<g>` elements don't support CSS transform-origin directly in all browsers, add `style={{ transformBox: 'fill-box', transformOrigin: 'center' }}` to the box group.

Track claimed boxes similarly: use a `claimedBoxesRef` to detect newly claimed boxes and only animate new ones:
```typescript
const claimedBoxesRef = useRef<Set<string>>(new Set());
// In box render:
const boxKey = `box-${r}-${c}`;
const isNewlyClaimed = owner !== null && !claimedBoxesRef.current.has(boxKey);
if (owner !== null) claimedBoxesRef.current.add(boxKey);
```

Apply `className={isNewlyClaimed ? 'animate-box-claim' : ''}` and `style={{ transformBox: 'fill-box', transformOrigin: 'center' }}` to the box `<g>`.
